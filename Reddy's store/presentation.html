Technology Stack

Libraries & Tools We Used

Every technology was chosen with purpose. Here's what powers this project under the hood.

Python 3.10+
Our primary backend language. Handles all server-side logic, data processing, and API routing. Chosen for its readability and rich ecosystem.
Used for: API logic, data models, file I/O, UUID generation, date handling
FastAPI
Modern, high-performance Python web framework for building APIs. Supports automatic request validation, async handlers, and auto-generated docs.
Used for: REST API endpoints, request routing, file serving, error handling
Uvicorn (ASGI Server)
Lightning-fast ASGI web server that runs our FastAPI application. Supports hot-reload during development for instant code changes.
Used for: Hosting the server on localhost:8000, hot-reload dev mode
Pydantic
Data validation library using Python type annotations. Automatically validates incoming checkout requests, shipping details, and cart items.
Models: CartItem, ShippingDetails, CheckoutRequest + EmailStr validation
HTML5
Semantic markup for the entire storefront — navigation, product grid, cart drawer, checkout modal, success screens, and order history.
Structure: 286 lines of clean, semantic HTML with modals & drawers
Vanilla CSS3
1,278 lines of hand-crafted CSS. Custom design system with glassmorphism, CSS variables, animations, responsive grids, and a rangoli-pattern background.
Techniques: backdrop-filter, CSS variables, @keyframes, SVG data-URI patterns
Vanilla JavaScript (ES6+)
610 lines of client-side logic. Handles product fetching, real-time search/filter, cart state with localStorage persistence, checkout form submission, and invoice downloads.
Features: Fetch API, DOM manipulation, localStorage, async/await
JSON File Database
A lightweight, file-based database (orders.json) for persisting order records. No external database engine needed — perfect for academic demonstrations.
Storage: orders.json — auto-created on first checkout
External Libraries
Google Fonts (Outfit + Inter) for modern typography. Font Awesome 6.4 for a rich icon system across the UI. Unsplash for product imagery.
CDN: fonts.googleapis.com, cdnjs.cloudflare.com, unsplash

Backend APIs

REST API Endpoints

Our FastAPI backend exposes 6 clean REST endpoints. Each one is validated, documented, and purpose-built.

Method Endpoint Purpose Returns
GET / Serve the main HTML page index.html (FileResponse)
GET /api/products Fetch all 10 products JSON array of product objects
GET /api/products/search Search, filter & sort products Filtered JSON (q, category, price, sort)
POST /api/checkout Process order & save to DB Order confirmation + totals
GET /api/orders Retrieve all past orders Complete order history array
GET /api/invoice/{order_id} Generate & download invoice Text file stream (downloadable)

User Journey

How The App Works

Follow the complete user journey — from landing on the homepage to downloading a paid invoice. Every step is real.

🌐 Visit Site
🔍 Browse & Filter
🛒 Add to Cart
🎟️ Apply Coupon
💳 Checkout
📄 Get Invoice
1. User opens localhost:8000
FastAPI serves index.html via the GET / route. CSS and JS are loaded as static files. The product catalog is fetched from /api/products/search.
2. Browsing & Filtering Products
Users can search by name, filter by category tabs, adjust the price range slider, and sort products — all instantly via JS Fetch API calls to the backend.
3. Cart Management
Products are added to a sliding cart drawer. Quantities can be adjusted. Cart state is persisted to localStorage so it survives page refreshes.
4. Coupon Codes & Pricing
Three working coupons: COLLEGE20 (20% off), SAMRUDHI15 (15% off), FREESHIP (free shipping). Tax is auto-calculated at 8%. Shipping is free above $150.
5. Secure Checkout & Payment
A modal collects shipping + card details. On submit, JS sends a POST /api/checkout request. FastAPI validates with Pydantic, calculates totals, generates an order ID, and saves to orders.json.
6. Invoice & Order History
Success screen offers invoice download — calling GET /api/invoice/{id} which generates a formatted text receipt. Order history modal reads all past transactions from the database.
Key Features

What Makes It Special

This isn't a basic CRUD app. Here are the features that make it feel like a real product.

Glassmorphic UI Design
Frosted glass panels, ambient glow orbs, smooth hover animations, rangoli SVG background patterns, and a curated Indian color palette — all in pure CSS. No frameworks, no Tailwind.
Real-Time Search & Filters
Instant search-as-you-type queries, category tabs (Phones, Laptops, Audio, Wearables, Accessories), price range slider, and multi-column sorting — all powered by backend API queries.
Persistent Shopping Cart
Cart slides in as a sleek drawer. Add, remove, adjust quantities. State is saved to localStorage — refresh the page, your cart stays intact. Subtotal & shipping auto-update.
Academic Coupon System
Three real coupon codes that work both client-side (for preview) and server-side (for actual calculations). Backend is the source of truth — no client-side cheating!
Dynamic Invoice Generation
Backend generates formatted text invoices on-the-fly using StreamingResponse. Each invoice includes order ID, customer info, itemized list, discounts, tax, and grand total.
Order History Archive
Every order is logged to orders.json. Users can view a complete transaction history modal and re-download any past invoice — simulating a real order management system.
Development Roadmap

How We Built It

The honest, step-by-step journey from an empty folder to a working e-commerce platform. Here's our actual build process.

Phase 1 — Planning & Research
Decided on the tech stack. Chose FastAPI over Flask for automatic validation. Sketched out the product catalog schema and API endpoints on paper. Studied glassmorphism & Indian color aesthetics.
Phase 2 — Backend First

Phase 4 — CSS Design System
Created 1,278 lines of CSS — custom variables for the Indian color palette, glassmorphic panels, responsive grid layouts, smooth animations, custom scrollbars, and the rangoli SVG background.
Phase 5 — JavaScript Interactivity
Wrote 610 lines of JS — Fetch API integration, real-time search/filter, cart state management with localStorage, coupon validation, checkout form handling, and invoice download triggering.
Phase 6 — Testing & Polish
Tested end-to-end flow: product browsing → cart → checkout → invoice. Verified coupon logic, edge cases (empty cart, invalid codes). Added micro-animations, loading states, and error messages.
